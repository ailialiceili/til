# AIを使った開発の進め方

AI開発には作法がある。

## 心がけ

- コードを直接直さない
- できるだけコードは書かないように（AIに任せる）
- 最初は小規模に作り、のちに追加機能を実装する
- 新しいことをやるときは**新しいチャット**を始める

## 手順（自開発編）

### 準備：要件定義書の作成

AIに要件定義書を作ってもらう。

**ステップ①：要件定義の壁打ち**

```
ToDoアプリを作りたい
まずは要件定義をしたい
不明点を潰していきたいので、わからないところをどんどん質問してください

・Windowsで動かす
・Pythonで開発する
```

AIが返してきた質問をさらにつぶしていく。

**ステップ②：要件定義書（完全版）の出力**

```
要件定義書として保存したいので不要な文章は全部削除
最初と最後の挨拶は不要
純粋な要件定義書の文書として出力して
```

### ①リポジトリ作成

- GitHubでリポジトリを作成する
- `.gitignore` を作成して一度コミット・プッシュ
- **リポジトリ名はハイフンを使うこと・すべて小文字！！**
  - （空白にすると自動でハイフンがつく）

### ②要件定義書をリポジトリに保存

`docs/requirements.md` を作成し、要件定義書をコピペ → コミット・プッシュ

### ③コード生成

VS Code で要件定義書のフォルダを `@` で指定し、コードを作ってもらう。

```
@/docs/requirements.md
この要件定義書の内容を基に開発を開始してください
pip install 必要なものは /requirements.txt にまとめてください
```

### ④アプリの起動

`run_app.bat` と `run_setup.bat` をリポジトリ内に置いて起動する。

**初回のみ、以下の手順でセットアップ：**

1. `run_setup.bat` をダブルクリック
2. 「完了」と出ればOK
3. `run_app.bat` をダブルクリックで実行！

### ⑤README作成・最終コミット

動作が大丈夫そうであれば、利用者向けREADMEを書いてもらい、最終コミット。

```
README.md を作成してください

利用者向けの内容を作成して
開発者向けの内容は不要
```
